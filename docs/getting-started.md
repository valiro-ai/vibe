# Getting Started

This guide walks you through setting up vibe in your repository.

## Prerequisites

- Go 1.21+ (for building from source)
- Claude Code CLI (for AI commands)

## Installation

### Build from Source

```bash
git clone <repo>
cd vibe
go build -o vibe ./cmd/vibe/
```

### Add to PATH (optional)

```bash
# Add to your shell profile
export PATH="$PATH:/path/to/vibe"
```

## Initialize Your Repository

Run `vibe init` in your repository root:

```bash
./vibe init
```

This creates:

```
your-repo/
├── docs/
│   └── seps/
│       ├── 0000-sep-process.md    # Process documentation
│       └── SEP-TEMPLATE.md        # Template for new SEPs
└── .claude/
    └── commands/
        ├── sep-discuss.md         # Claude command: /sep-discuss
        ├── sep-plan.md            # Claude command: /sep-plan
        ├── sep-implement.md       # Claude command: /sep-implement
        ├── sep-new.md             # Claude command: /sep-new
        ├── sep-split.md           # Claude command: /sep-split
        ├── sep-status.md          # Claude command: /sep-status
        └── sep-suggest.md         # Claude command: /sep-suggest
```

## Create Your First SEP

```bash
./vibe sep new "User Authentication"
```

This creates `docs/seps/0001-user-authentication.md`:

```markdown
---
title: "User Authentication"
status: DRAFT
created: 2025-01-15
depends_on: []
areas: []
assigned: ""
---

# SEP-0001: User Authentication

## What & Why

[Explain what you're building and why you want it.]

## Done When

- [ ] [Acceptance criteria 1]
- [ ] [Acceptance criteria 2]

---

## Plan

*Generated by /sep-plan*

---

## Implementation Notes

*Added during /sep-implement*
```

## Fill in the SEP

Edit the SEP to define:

1. **What & Why** - Describe the feature and its value
2. **Done When** - List acceptance criteria (checkboxes)
3. **Areas** - List files/directories this SEP will modify

Example:

```yaml
areas:
  - internal/auth/*
  - api/routes/login.go
  - cmd/server/main.go
```

## Check Status

```bash
./vibe sep status
```

Output:

```
SEP Status
========================================

DRAFT (awaiting review):
  - SEP-0001: User Authentication (created 2025-01-15)

NEXT: Review SEP-0001 (editor approval needed before implementation)
```

## Get Editor Approval

An editor reviews the SEP and approves it:

```bash
./vibe sep update 0001 ACCEPTED
```

Now status shows:

```
ACCEPTED (ready for implementation):
  - SEP-0001: User Authentication (created 2025-01-15)

NEXT: Implement SEP-0001 with /sep-plan then /sep-implement
```

## Claim and Implement

As a pilot, claim the SEP and implement it:

```bash
# Claim the SEP (shares with team via git)
./vibe sep claim 0001 @yourname

# Create implementation plan
/sep-plan 0001

# Review the plan in the SEP, then implement
/sep-implement 0001
```

Claude will read the SEP, follow the plan, and implement the feature.
